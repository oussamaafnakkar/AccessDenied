# SecureBank Customer Service Procedures

**Document Version:** 2.1  
**Last Updated:** February 1, 2026  
**Department:** Customer Service  
**Classification:** Internal Use

## Purpose

This document outlines standard operating procedures for SecureBank customer service representatives when assisting customers through various channels.

## General Guidelines

### Customer Interaction Principles

1. **Professional Communication**: Always maintain a courteous and professional tone
2. **Active Listening**: Allow customers to fully explain their concerns
3. **Empathy**: Acknowledge customer frustrations and concerns
4. **Efficiency**: Resolve issues promptly while maintaining quality
5. **Security**: Verify customer identity before discussing account details

## Account Services

### Opening New Accounts

**Procedure:**
1. Verify customer eligibility (age 18+, valid ID, SSN)
2. Collect required information:
   - Full legal name
   - Date of birth
   - Residential address
   - Contact information
   - Employment details
3. Explain account types and features
4. Complete application in banking system
5. Initiate identity verification process
6. Set up initial deposit (minimum $25 for savings, $100 for checking)
7. Provide account number and routing information
8. Explain online banking setup

**Required Documents:**
- Government-issued photo ID (driver's license or passport)
- Social Security card or documentation
- Proof of address (utility bill, lease agreement)

### Password Reset

**Procedure:**
1. Verify customer identity using:
   - Last 4 digits of SSN
   - Account number
   - Date of birth
   - Security question answers
2. Generate temporary password
3. Send via secure channel (email or SMS to registered contact)
4. Require password change on next login
5. Document reset in customer interaction log

**Security Notes:**
- Never send passwords via unencrypted email
- Temporary passwords expire in 24 hours
- Maximum 3 reset attempts per day

### Account Balance Inquiries

**Procedure:**
1. Verify customer identity (see Authentication section)
2. Access account in banking system
3. Provide current balance information:
   - Available balance
   - Current balance
   - Pending transactions
4. Explain difference between available and current balance if asked
5. Offer to email detailed statement

**Privacy Note:** Never disclose balance information without proper authentication

## Transaction Services

### Funds Transfer

**Procedure:**
1. Verify sufficient funds in source account
2. Confirm transfer details:
   - From account
   - To account
   - Amount
   - Transfer date (immediate or scheduled)
3. Verify recipient information for external transfers
4. Execute transfer in system
5. Provide confirmation number
6. Set up alerts if requested

**Limits:**
- Internal transfers: $50,000 per transaction
- External transfers: $10,000 per day
- Wire transfers: $100,000 per transaction (requires additional verification)

### Dispute Resolution

**Procedure:**
1. Listen to customer's concern about transaction
2. Gather transaction details:
   - Date
   - Amount
   - Merchant name
   - Authorization code
3. Determine dispute type:
   - Unauthorized transaction
   - Billing error
   - Product/service not received
4. Initiate dispute in system
5. Provide provisional credit if applicable (within 10 business days)
6. Explain investigation timeline (45-90 days)
7. Send dispute confirmation letter

**Provisional Credit Guidelines:**
- ATM/Debit card: Within 10 days
- Credit card: Within 2 billing cycles
- Amount: Full disputed amount

## Technical Support

### Online Banking Access Issues

**Common Issues and Solutions:**

1. **Forgot Username**
   - Verify identity
   - Username is typically email address
   - Can be found on account statements

2. **Forgot Password**
   - Follow password reset procedure
   - Send temporary password to registered email/phone

3. **Account Locked**
   - Occurs after 5 failed login attempts
   - Unlock after identity verification
   - Expires automatically after 30 minutes

4. **Two-Factor Authentication Issues**
   - Verify phone number on file is current
   - Resend authentication code
   - Update contact information if needed

### Mobile App Support

**Troubleshooting Steps:**
1. Verify app is updated to latest version
2. Clear app cache
3. Uninstall and reinstall if persistent issues
4. Check device compatibility
5. Verify internet connection

## Fraud Prevention

### Suspicious Activity

**Red Flags:**
- Unusual transaction patterns
- Large withdrawals or transfers
- Changes to contact information followed by transfers
- Requests to bypass security procedures
- Customer seems coerced or under duress

**Action Steps:**
1. Document observations
2. Verify customer identity thoroughly
3. Ask security questions
4. If fraud suspected:
   - Do not alert customer
   - Escalate to fraud department immediately
   - Place temporary hold on account if necessary
5. File Suspicious Activity Report (SAR) if required

### Lost or Stolen Cards

**Procedure:**
1. Immediately deactivate card in system
2. Review recent transactions for fraud
3. Initiate dispute for unauthorized charges
4. Order replacement card:
   - Rush delivery (2-3 days): $25 fee
   - Standard delivery (7-10 days): Free
5. Provide temporary card number for urgent needs
6. Update recurring payments if needed

## Escalation Procedures

### When to Escalate

Escalate to supervisor when:
- Customer requests to speak with manager
- Issue requires override authority
- Complex technical problems
- Potential fraud or security concerns
- Customer is extremely dissatisfied
- Issue exceeds your knowledge or authority

### Escalation Process

1. Inform customer you're escalating to supervisor
2. Provide supervisor with complete context
3. Brief summary of:
   - Customer name and account
   - Issue description
   - Steps already taken
   - Customer's desired resolution
4. Warm transfer (stay on line) when possible

## Performance Metrics

### Key Performance Indicators (KPIs)

- **First Call Resolution**: Target 85%
- **Average Handle Time**: Target 5-7 minutes
- **Customer Satisfaction**: Target 90%+
- **Call Abandonment Rate**: Target <5%
- **Quality Assurance Score**: Target 95%+

### Quality Standards

All interactions must:
- Begin with professional greeting
- Include identity verification
- Document all actions in CRM
- End with confirmation customer's needs are met
- Offer additional assistance

## Compliance

### Regulatory Requirements

All customer service representatives must:
- Complete annual compliance training
- Follow Bank Secrecy Act (BSA) requirements
- Report suspicious activity
- Maintain customer privacy (GLBA)
- Document all customer interactions

### Data Privacy

**NEVER:**
- Share customer information with unauthorized parties
- Access accounts without business need
- Discuss customer information in public areas
- Use personal devices for customer data
- Write down passwords or PINs

## Emergency Procedures

### System Outage

1. Inform customer of outage
2. Provide estimated restoration time
3. Offer alternative service methods:
   - ATM access
   - Branch visit
   - Phone banking
4. Document customer contact for follow-up
5. Escalate if critical transaction needed

### Security Breach

1. Follow incident response plan
2. Alert security team immediately
3. Do not discuss details with customers
4. Direct questions to public relations
5. Document all related interactions
